# 「股神俱樂部」APP 產品需求文件（MVP）

## 產品概述
- **目標**：提供一個統整台股達人操作紀錄與互動的社群平台，讓一般用戶能在單一介面追蹤多位達人的房間、比較績效並參與討論，達人則可藉由優化操作績效累積聲量與轉換潛在商業價值。
- **裝置／設計基調**：iPhone 15（390 × 844 pt）、深色模式、極簡扁平＋8 pt 網格、Large 圓角、主色 #4ADE80、Feather Icons、字體 SF Pro（headline-4／body-1）。

## 目標用戶與價值
- **台股達人**：可創建房間、紀錄每日操作、與加入者互動，藉此展現績效、累積粉絲與商業轉換機會。
- **一般用戶**：可瀏覽並加入房間，查看達人操作與留言互動，縮短在多個群組間切換的時間，快速比較不同達人操作策略。

## MVP 成功指標
- DAU ≥ 5,000（上線後第 3 個月）
- 30 日留存率 ≥ 25 %
- 達人平均日發文（操作記錄）≥ 1 則
- 房間數量 ≥ 50（上線後第 3 個月）
- 會員人數 ≥ 100（上線後第 3 個月）

## 核心模組與功能
### 1) 會員／登入
- 首次開啟顯示登入頁：「歡迎進入股神俱樂部」標題＋「登入」副標。
- 欄位：Email、密碼；底部按鈕：註冊、登入。
- 註冊欄位：Email、電話、暱稱、密碼、確認密碼（免驗證）。註冊後即成為會員並可登入。
- 完成登入進入首頁（房間總覽），不區分一般用戶與達人。

### 2) 導航列（全域）
- 底部分頁：Screen 01 房間總覽（Default）｜Screen 02 我創建的房間｜Screen 03 我加入的房間。
- Header 左：標題「股神俱樂部」，點擊返回房間總覽。
- Header 中右：按鈕「創建房間」→ 彈出創建房間 popup。
- Header 右：頭像按鈕 → 前往個人主頁（Screen 05）。

### 3) 房間總覽（Screen 01，首頁）
- 卡片列出所有已創建房間（不含自己創建的）。
- 卡片欄位：左側房間名稱；右側從上而下：創建者暱稱、房間人數、操作週期、近期操作標的（依日期新→舊取前三筆股票代號＋名稱）。
- 卡片按鈕：右下「加入房間」。加入後顯示「加入成功」toast；若已加入則顯示「進入房間」→ 前往房間頁-訪客。
- 排序：頁面右上按鈕，可依「房間人數 多→少」排序。

### 4) 創建房間（Popup）
- 標題「創建房間」。
- 必填：房間名稱（文字/數字）、操作週期（短線波段／中期波段／長期波段／價值投資）。
- 選填：房間介紹（文字/數字）。
- 送出後顯示「成功創建」toast，並自動跳轉到我創建的房間（Screen 02）。

### 5) 我創建的房間（Screen 02）
- 空狀態：顯示「您尚未創建房間」。
- 頁面資訊：上方房間名稱，右側顯示房間人數；左側「新增操作記錄」按鈕。
- 空紀錄：顯示「尚未新增操作記錄」。
- 操作記錄卡片欄位：股票代號、股票名稱、股票張數、日期、操作（買進／加碼／減碼／賣出）、操作說明。
- 功能：
  - 新增操作記錄（右上按鈕 → popup，欄位含：股票代號(數字)、股票名稱(文字)、股票張數(數字)、日期(預設今日，行事曆選取)、操作下拉、操作說明；送出後顯示在頁面）。
  - 編輯操作記錄（卡片右側編輯按鈕 → 同新增欄位可更新）。
  - 刪除操作記錄（卡片右側刪除按鈕 → popup「確認刪除」）。
  - 我要留言（卡片下方按鈕 → popup 輸入框可文字/數字，送出後顯示於卡片留言區）。
  - 移除房間（頁面左下「移除房間」 → popup「確認移除」，移除後回到空狀態）。

### 6) 我加入的房間（Screen 03）
- 空狀態：顯示「您尚未加入任何房間」。
- 卡片格式與房間總覽一致；按鈕為「進入房間」→ 前往房間頁-訪客。
- 排序：可依房間人數多→少排序。

### 7) 房間頁 - 訪客（Screen 04）
- 上方顯示房間名稱與房間人數。
- 若無操作記錄：顯示「尚未新增操作記錄」。
- 操作記錄卡片欄位同達人版；下方留言區顯示使用者留言。
- 功能：我要留言（與創建者同）、退出房間（左下按鈕 → popup「確認退出」，返回我加入的房間）。

### 8) 個人主頁（Screen 05）
- 顯示使用者暱稱。
- 創建的房間：列出房間名稱；若無顯示「無」。
- 加入的房間：列出房間名稱與創建者暱稱；若無顯示「無」。

## 體驗細節
- Toast 文案：加入成功、成功創建、確認刪除、確認移除、確認退出等行為的即時反饋。
- 欄位驗證：僅必要必填檢查，不做進階校驗（依需求）。
- 排序狀態：按鈕需呈現目前排序（預設依建立時間新→舊），切換後即時更新列表。
- 深色模式：背景採深色底，主色 #4ADE80 用於主按鈕與強調元素，保持高對比；卡片與彈窗採 Large 圓角，8 pt 間距網格。

## 里程碑（建議）
1. **帳號與房間 CRUD 架構**：會員註冊／登入、房間創建與列表；完成 Screen 01～05 基礎導覽與狀態。
2. **操作記錄與留言互動**：新增／編輯／刪除操作記錄、留言串接、房間加入／退出流程完成。
3. **UI 細節優化**：排序、toast、深色模式與元件一致性調整，導覽流暢度驗證。
4. **MVP 上線指標追蹤**：事件埋點以對應 DAU、留存、房間數、操作發文數等目標。
